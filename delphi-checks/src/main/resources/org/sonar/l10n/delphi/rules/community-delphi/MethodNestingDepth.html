Why is this an issue?

Deeply nested subroutines make for code that is difficult to navigate and understand. While subroutines can be helpful for small units of work specific to a particular method, their ambiguous syntax and non-linear control flow make them confusing when containing large amounts of functionality.

How to fix it

Extract the nested subroutines out into one or more independent methods:

procedure DoThing;
  procedure DoSubThing;
    procedure DoSubSubThing;
    begin
      // ...
    end;
  begin
    // ...
  end;
begin
  // ...
end;

procedure DoSubThing;
  procedure DoSubSubThing;
  begin
    // ...
  end;
begin
  // ...
end;

procedure DoThing;
begin
  // ...
end;
