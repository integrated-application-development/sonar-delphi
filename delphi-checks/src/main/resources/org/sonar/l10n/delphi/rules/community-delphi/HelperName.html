Why is this an issue?

Code that follows a consistent naming convention is self-documenting. In Delphi, all class or record helper names should:

  • Be in PascalCase
  • Start with a consistent prefix
  • Contain the name of the extended type
  • End with Helper

The default prefix is T, but some projects may use another prefix to separate their helpers from those of other projects. This can make code spanning several projects more easily understandable.

How to fix it

Rename the class or record helper to follow the convention.
