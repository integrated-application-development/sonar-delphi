Why is this an issue?

Methods with many subroutines make for code that is difficult to navigate and understand. While nested methods can be helpful for small units of work specific to a particular method, their easily missed syntax and non-linear control flow make them confusing when containing large amounts of functionality.

How to fix it

Consider if the subroutines can be inlined to the body of the main method:

procedure DoThing;
  procedure DoSubThing;
  begin
    Log('Hello!');
  end;
  // ...
begin
  DoSubThing;
  // ...
end;

procedure DoThing;
begin
  Log('Hello');
  // ...
end;

Otherwise, extract the subroutines out into one or more independent methods:

procedure DoThing;
  procedure DoSubThing;
  begin
    // ...
  end;
  // ...
begin
  // ...
end;

procedure DoSubThing;
begin
  // ...
end;

procedure DoThing;
begin
  // ...
end;
