Why is this an issue?

Having consecutive const sections is useless and negatively impacts readability. Combining the sections into one larger section is more concise and easier to understand.

This rule also picks up consecutive resourcestring sections.

How to fix it

Combine the const or resourcestring sections together:

const
  CFoo = 5;
  CBar = 'abc';
const
  CBaz = 24.4;

const
  CFoo = 5;
  CBar = 'abc';
  CBaz = 24.4;
