Why is this an issue?

Having unused methods adversely affects code readability, as their presence is confusing. Code that is never used should be removed to keep codebases clean and maintainable.

Note that this rule excludes published methods, as these are often accessed via reflection or used by the Delphi IDE for design-time configuration. Methods marked as override and methods implementing an interface are also excluded.

How to fix it

Remove the unused method.
