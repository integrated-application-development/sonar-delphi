Why is this an issue?

Empty methods are usually superfluous, and should be removed to prevent unexpected behaviour.

Some valid empty methods could include:

  • A virtual method that is intended to be overridden
  • An override method of a non-empty virtual method
  • A method that exists only to implement an interface

In these cases, a comment should be added to the method body to explain why the method is blank.

How to fix it

If the empty method is an omission, remove it.

If the method should be empty, add a nested comment explaining why the method is blank:

Noncompliant code example

procedure MyEmptyMethod; // Noncompliant
begin
end;

Compliant solution

procedure MyEmptyMethod;
begin
  // Empty because I really want it to be!
end;
