Why is this an issue?

Visibility specifier directives (private, public, etc.) should use the same indentation as the type name that they are declared under. This allows developers to easily differentiate between fields, methods, and properties of different visibilities.

Please note that this rule checks for an exact match on trailing whitespace - mixed indentation may yield unexpected results.

How to fix it

Indent the visibility specifier at the same level as the enclosing type declaration:

  type
    TFoo = class(TObject)
      private
      procedure Bar;
      public
      public constructor Create;
    end;

  type
    TFoo = class(TObject)
    private
      procedure Bar;
    public
      constructor Create;
    end;
