Why is this an issue?

Subroutines cannot be used as procedural values.

This will normally cause a compilation error, but such errors can be bypassed by using the address of the subroutine with @.

In this scenario, the code generated by compiler will have undefined behavior.

How to fix it

Do not use @ to get the address of a subroutine. If a method pointer is required, convert the subroutine into a top-level method.
