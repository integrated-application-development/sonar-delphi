Why is this an issue?

Unnecessarily using control flow intrinsics makes the code harder to read and understand with no benefit. These statements appear to jump in the code, however, the jump is to the location that would be next anyway.

How to fix it

Remove the redundant jump:

procedure Example(Foo: TFoo);
begin
  while Condition do begin
    Foo;
    Continue; // Noncompliant
  end;
end;

procedure Example(Foo: TFoo);
begin
  while Condition do begin
    Foo;
  end;
end;
