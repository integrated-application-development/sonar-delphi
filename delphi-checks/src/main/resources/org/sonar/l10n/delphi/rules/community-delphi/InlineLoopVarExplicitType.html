Why is this an issue?

Omitting the type for inline loop variable declarations has a few downsides, including:

  • The lack of type information can make code harder to follow and understand.
  • Type inference can lead to unexpected types being inferred, particularly for integers.
  • Wrapper types that rely on implicit type conversions will not have those conversions invoked.

How to fix it

Explicitly specify the loop variable's type:

procedure Example;
begin
  for var I := 1 to 100 do begin
    Continue;
  end;
end;

procedure Example;
begin
  for var I: Integer := 1 to 100 do begin
    Continue;
  end;
end;
