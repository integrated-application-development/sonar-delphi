Why is this an issue?

Code that follows a consistent naming convention is self-documenting. In Delphi, all record names should be in PascalCase and be prefixed with a consistent prefix.

The default prefix is T, but some projects may use another prefix to separate their records from those of other projects. This can make code spanning several projects more easily understandable.

How to fix it

Rename the record to follow the convention.
